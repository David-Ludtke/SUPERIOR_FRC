sfrc::util::RobotArray::PushBackArray:

void PushBackArray(T* array, size_t length)

as the name says it pushes back
a c style array into the RobotArray
takes in a pointer to the array and
the length of said array
the array is added to the end of the RobotArray
this function does not check the validity of the pointer
nor the length provided the caller is responsible for ensuring
the length is non 0 and the pointer is valid
this function may cause undefined behavior if
invalid parameters are provided
the function may fail the program if memory cannot be allocated

Examples:
  • 
    sfrc::Util::RobotArray<int> myArray = sfrc::Util::RobotArray<int>::Create();
    int arr[] = {1, 2, 3, 4, 5};
    myArray.PushBackArray(arr, 5);
    //myArray now contains {1, 2, 3, 4, 5};
                    
  • 
    sfrc::Util::RobotArray<std::string> strArray = sfrc::Util::RobotArray<std::string>::OfVec({"hello", "world"});
    const char* arr[] = {"foo", "bar", "baz"};
    strArray.PushBackArray(arr, 3);
    //strArray now contains {"hello", "world", "foo", "bar", "baz"};
